  <action>
    The user must complete these steps manually. Claude cannot fork repos, create Neon accounts, or set Vercel env vars on external projects.

    **Step 1 — Create Neon Postgres database (per D-03):**
    1. Sign up at https://neon.tech (free, no credit card)
    2. Create a new project (any name, e.g., "umami-analytics")
    3. Choose region closest to expected visitors (us-east-2 is a good default)
    4. Create a database named `umami`
    5. Copy the **pooled** connection string (contains `-pooler` in hostname):
       `postgresql://<user>:<password>@ep-<id>-pooler.<region>.aws.neon.tech/umami?sslmode=require`

    **Step 2 — Fork and deploy Umami (per D-01, D-02):**
    1. Go to https://github.com/umami-software/umami and click Fork
    2. Go to https://vercel.com/new and import the forked umami repo
    3. **Before deploying**, add environment variables:
       - `DATABASE_URL` = the Neon pooled connection string from Step 1
       - `APP_SECRET` = generate with `openssl rand -base64 32`
    4. Deploy the project
    5. Wait for deployment to complete — Umami will run Prisma migrations on first startup

    **Step 3 — Secure credentials (per D-05):**
    1. Visit the deployed Umami URL (e.g., `https://your-umami.vercel.app`)
    2. Log in with default credentials: username `admin`, password `umami`
    3. Go to Settings -> Profile and change the password IMMEDIATELY
    4. Change the username from `admin` to something personal

    **Step 4 — Add website to Umami:**
    1. In Umami dashboard, go to Settings -> Websites -> Add website
    2. Name: "MSizzle" (or preferred name)
    3. Domain: `msizzle.com`
    4. Copy the Website ID (UUID) — this is needed for `NEXT_PUBLIC_UMAMI_WEBSITE_ID`

    **Step 5 — Set main site env vars (production only, per D-09):**
    1. Go to the main MSizzle Vercel project settings -> Environment Variables
    2. Add `NEXT_PUBLIC_UMAMI_WEBSITE_ID` = the UUID from Step 4 (Production environment only)
    3. Add `NEXT_PUBLIC_UMAMI_URL` = the Umami instance URL, e.g., `https://your-umami.vercel.app` (Production environment only)
    4. Do NOT add these to Preview or Development environments — this prevents dev traffic pollution
    5. Trigger a new deployment of the main site to pick up the new env vars

    **Step 6 (Optional) — Custom subdomain (per D-04):**
    1. In your DNS provider, add CNAME record: `analytics.msizzle.com` -> `cname.vercel-dns.com`
    2. In the Umami Vercel project settings, add `analytics.msizzle.com` as a custom domain
    3. Wait for DNS propagation (5 min to 48 hours)
    4. Verify: `nslookup analytics.msizzle.com` should resolve
    5. If custom domain is configured, update `NEXT_PUBLIC_UMAMI_URL` in main site to `https://analytics.msizzle.com`
  </action>
