  1. .next/static/chunks/ (from next build)
  2. .vercel/output/static/_next/static/chunks/ (from vercel build)
Plus a broader .vercel/output/ sweep for function bundles and prerendered HTML.

Leak semantics:
  - Client chunks (Part A): ANY hit (including NOTION_TOKEN variable name) = LEAK per D-14.
  - Broader .vercel/output/ sweep (Part B): only secret VALUES (secret_, ntn_, DB-ID literal)
    count as LEAK. The variable NAME NOTION_TOKEN is expected in server function bundles
    (process.env reads from Edge runtime) and is not a leak. D-14 treats client chunks as
    the blocking gate; the broader sweep is a belt-and-suspenders check for literal secret
    values that escaped into server bundles.
