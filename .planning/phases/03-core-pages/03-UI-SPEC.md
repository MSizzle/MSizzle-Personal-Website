## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, tag pill vertical padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, tag pill horizontal padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, nav horizontal padding |
| lg | 24px | `p-6` / `gap-6` | Card internal padding, section horizontal padding |
| xl | 32px | `p-8` / `gap-8` | Card grid gap, content stack gap |
| 2xl | 48px | `py-12` / `gap-12` | Section padding top/bottom |
| 3xl | 64px | `py-16` / `gap-16` | Page-level section breaks |
| 4xl | 96px | `py-24` | Hero section top padding (offset for fixed nav) |

Exceptions:
- Fixed nav height: 64px (`h-16`) â€” touch-safe, consistent offset for scroll anchoring
- Mobile nav hamburger touch target: 44px minimum (`min-h-[44px] min-w-[44px]`) â€” WCAG 2.5.5
- Full-viewport hero and narrative sections: `min-h-screen` â€” per D-01, not a spacing token

## Typography

All sizes are pixel equivalents of Tailwind v4 rem values at 16px base.

**Exactly 2 weights: Regular (400) and Semibold (600).** No other weights are used anywhere in the system.

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height | Usage |
|------|------|----------------|--------|--------------|-------------|-------|
| Display | 60px (sm: 72px) | `text-6xl sm:text-7xl` | Semibold (600) | `font-semibold` | 1.1 | Hero h1 "Hey, I'm Monty." â€” source: existing `page.tsx` |
| Heading | 30px (sm: 48px) | `text-3xl sm:text-5xl` | Semibold (600) | `font-semibold` | 1.2 | Section h2 titles, page titles, card titles, section subheads â€” source: existing `page.tsx` |
| Body | 16px | `text-base` | Regular (400) | `font-normal` | 1.6 | Prose content, blog post body, about page narrative |
| Label | 14px | `text-sm` | Regular (400) | `font-normal` | 1.4 | Tag pills, reading time, date stamps, nav links |

**Card titles and section subheads** that previously used the removed Subheading role use `text-xl font-semibold` (a Heading-scale variant, not a separate named role).

**Letter spacing:** Use `tracking-tight` (`letter-spacing: -0.025em`) on Display and Heading sizes. Body and labels: default tracking.

**Prose content** (blog posts, about page): rendered inside `prose prose-neutral dark:prose-invert` from `@tailwindcss/typography`. Already installed in Phase 2.

## Color

The existing `globals.css` establishes the base token palette. This contract extends it.

### Base Tokens (already in globals.css)

| CSS Variable | Light Value | Dark Value | Role |
|---|---|---|---|
| `--bg` | `#fafafa` | `#0a0a0a` | Dominant background |
| `--fg` | `#0a0a0a` | `#fafafa` | Primary foreground text |

### Extended Tokens to Add in globals.css

Add the following to `:root` and `.dark` blocks:

```css
:root {
  --bg: #fafafa;
  --fg: #0a0a0a;
  --bg-secondary: #f0f0f0;   /* cards, nav bg, code blocks */
  --fg-muted: #737373;        /* neutral-500 â€” subtitles, dates, reading time */
  --border: #e5e5e5;          /* neutral-200 â€” card borders, section dividers */
  --accent: #2563eb;          /* blue-600 â€” reserved for specific elements below */
}

.dark {
  --bg: #0a0a0a;
  --fg: #fafafa;
  --bg-secondary: #171717;    /* neutral-900 â€” cards, nav bg */
  --fg-muted: #a3a3a3;        /* neutral-400 â€” subtitles, dates, reading time */
  --border: #262626;          /* neutral-800 â€” card borders, section dividers */
  --accent: #3b82f6;          /* blue-500 â€” slightly lighter in dark mode */
}
```

### 60/30/10 Color Split

| Role | Value | Tailwind Equivalent | Usage |
|------|-------|---------------------|-------|
| Dominant (60%) | `--bg` `#fafafa` / `#0a0a0a` | `bg-background` | Page background, hero section, about page body area |
| Secondary (30%) | `--bg-secondary` `#f0f0f0` / `#171717` | `bg-neutral-100 dark:bg-neutral-900` | Project cards, nav background (on scroll), footer, tag pills (inactive), code blocks |
| Accent (10%) | `--accent` `#2563eb` / `#3b82f6` | `text-blue-600 dark:text-blue-500` | See reserved list below |
| Destructive | `#ef4444` | `text-red-500` | Not applicable in Phase 3 (no destructive actions) |

**Accent reserved for â€” explicit list (accent used nowhere else):**
1. Primary CTA button background on hero ("View My Work")
2. Active tag pill background on blog filter
3. Navigation active route underline indicator
4. Hyperlinks within prose content (blog posts, about page)
5. Newsletter CTA subscribe button

**Muted text color (`--fg-muted`) usage:**
- Blog post dates, reading time estimates
- Project card descriptions (secondary line)
- Footer copyright text
- About page section metadata

## Component Inventory

Components to build in this phase with their visual contract:

### Navigation (navigation.tsx)

- Fixed top bar, `z-50`, full width, height 64px (`h-16`)
- Background: transparent when at top of page, `bg-background/80 backdrop-blur-sm border-b border-border` after scroll
- Left: "Monty Singer" in `text-base font-semibold tracking-tight`
- Center/right: nav links in `text-sm`, active route has accent underline (`border-b-2 border-accent`)
- Right: ThemeToggle (existing component)
- Mobile: hamburger button uses `Menu` icon from lucide-react at 24px. The button element must carry `aria-label="Open navigation menu"` (and `aria-label="Close navigation menu"` when open). Nav links in full-screen drawer, each link 48px tall minimum touch target.
- Transition: `transition-all duration-200` on background opacity change

### Project Card (project-card.tsx)

- Background: `bg-secondary` (uses `--bg-secondary`)
- Border: `border border-border rounded-xl`
- Padding: 24px (`p-6`)
- Thumbnail: 16:9 aspect ratio, `object-cover`, `rounded-lg`, full card width
- Title: `text-xl font-semibold` below thumbnail, 16px margin top
- Description: `text-sm text-muted-foreground` (maps to `--fg-muted`), 8px margin top, 3-line clamp (`line-clamp-3`)
- External link: "View Project" label with `ExternalLink` icon (16px), accent color text, 16px margin top
- Hover: `hover:border-accent/50 transition-colors duration-200` â€” no animation sequences (Phase 4 handles hover-reveal)

### Tag Pills (tag-filter.tsx)

- Inactive pill: `bg-neutral-100 dark:bg-neutral-800 text-sm px-3 py-1 rounded-full`
- Active pill: `bg-accent text-white text-sm px-3 py-1 rounded-full`
- "All" pill: same styles as active/inactive, clears filter when clicked
- Gap between pills: 8px (`gap-2`)
- Transition: `transition-colors duration-150`

### Links Page Cards (links page)

- Full-width card, `bg-secondary border border-border rounded-xl`
- Height: 64px minimum (`min-h-16`)
- Padding: 16px horizontal (`px-4`)
- Icon: 20px, left-aligned, `text-fg-muted`. Each icon-only element must carry an `aria-label` matching the link label (e.g., `aria-label="Twitter / X"`).
- Label: `text-base`, centered
- Arrow/external indicator: `ArrowUpRight` from lucide-react, right-aligned, `text-fg-muted`, `aria-hidden="true"`
- Gap between cards: 8px (`gap-2`). To visually separate cards at this tight gap, add `border-b border-border` to all cards except the last (`last:border-b-0`).
- Hover: `hover:bg-accent/5 hover:border-accent/30 transition-colors duration-150`

### Newsletter CTA Block (newsletter-cta.tsx)

- Full-width block below blog post content
- Background: `bg-secondary border border-border rounded-xl`
- Padding: 32px (`p-8`)
- Heading: `text-xl font-semibold`
- Copy: `text-sm text-muted-foreground mt-2`
- Input + button row: email input (`flex-1`) + submit button (accent background), 8px gap
- Implementation: placeholder form â€” submit logs to console, no real service wired (Claude's discretion, out of scope for Phase 3)

### Footer (footer.tsx)

- Background: `bg-secondary border-t border-border`
- Padding: 48px vertical, 24px horizontal (`py-12 px-6`)
- Social links: inline, 24px gap, icons at 20px from lucide-react. Each icon-only anchor must carry `aria-label` with the platform name (e.g., `aria-label="Twitter / X"`, `aria-label="LinkedIn"`, `aria-label="GitHub"`, `aria-label="Email"`).
- Copyright: `text-sm text-muted-foreground`
- Max width: `max-w-5xl mx-auto`

## Mobile Responsiveness Contract (DSGN-02)

Breakpoint: 375px (iPhone SE) is the minimum supported viewport.

| Component | Mobile Behavior |
|-----------|----------------|
| Navigation | Hamburger menu â€” full-screen nav drawer, `z-50` |
| Hero | Centered stack, h1 drops to `text-5xl` (60px), no horizontal overflow |
| Project grid | 1 column (`grid-cols-1`), full width cards |
| Blog list | Single column, full width |
| Tag pills | Horizontally scrollable row (`overflow-x-auto`) â€” no wrapping to prevent layout shift |
| Links page | Full width cards, vertical stack |
| Footer | Column layout, centered text |
| Max content width | `max-w-5xl mx-auto px-6` â€” 24px side padding on all viewports |

No horizontal overflow permitted at 375px on any page. All images must use `w-full` and `aspect-ratio` containment.

## Design Constraints (from ROADMAP.md Phase 3 risks)
